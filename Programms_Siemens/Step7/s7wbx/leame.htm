6         Informaciones para el uso de NCM S7

 

6.1      Observación sobre el archivo "Léame"

Lea con atención el archivo "Léame" antes de utilizar NCM S7. 

Este archivo contiene informaciones adicionales actuales relativas al producto, que complementan las explicaciones de los manuales.

 

El archivo "Léame" está disponible en el menú de Inicio tras finalizar la instalación. Inicio – SIMATIC – Informaciones sobre productos – NCM S7 – léame.htm.

Note that the software in MS Windows Vista is no longer under "Start menu > SIMATIC"; it is now located under "Start menu > Program Files > Siemens Automation > SIMATIC". This was made necessary by the current architecture of MS Windows Vista. The information in the documentation and online help must be interpreted accordingly.
 

6.2      Configuración del hardware

 

6.2.1      Configurar el modo Maestro DP

 

6.2.1.1  Sistema Maestro DP en caso de CP 342-5 como Maestro DP

Si en el CP 342-5 DP se ha definido en el modo "Maestro DP" el sistema maestro DP con un esclavo DP sin módulo, no es posible poner en funcionamiento el CP. Por esta razón, en el caso de este maestro DP se tiene que configurar al menos un módulo para cada esclavo DP configurado.

 

6.2.1.2  CP342-5 como esclavo DP  con LSAP 56

En el caso de un esclavo DP, el LSAP 56 está ocupado para la función "Read_Inputs: leer datos de entrada por maestro Kl2". En tal caso ya no se permite operar con el LSAP 56 una conexión FDL Broadcast o Multicast configurada a mismo tiempo.

6.2.1.3  Interrupción de la comunicación DP al cargar enlaces FDL

Si con el sistema en funcionamiento (estado RUN) se cargan enlaces FDL, se interrumpe la comunicación DP durante ese tiempo. Tenga en cuenta esto en caso de aplicaciones DP críticas en cuanto al tiempo (vea al respecto también la nota en el párrafo "Solución" del punto siguiente).

 

6.2.1.4  Perfil de bus crítico en cuanto al tiempo en caso de desarrollo paralelo del modo DP y enlaces FDL

Si opera en la subred PROFIBUS(L2) simultáneamente enlaces FDL y sistemas maestros DP, el perfil de parámetros de bus "Estándar" puede provocar un comportamiento crítico en cuanto al tiempo en los sistemas maestros DP. Si se presenta una carga elevada del bus debido a transmisiones vía enlaces FDL, pueden quedar excluidos esporádicamente esclavos DP de la transmisión, debido a que reacciona el límite de tiempo de supervisión de la respuesta ajustado en el maestro DP. Esto puede influir en el comportamiento de tiempo real de una aplicación crítica en cuanto al tiempo.

 

Solución:

Posibilidad 1: utilice el perfil "Universal".

Posibilidad 2: incremente el tiempo de retardo de DP en el cuadro de diálogo de propiedades para el CP. Con esto aumenta la ventana de tiempo disponible para la comunicación FDL. El ajuste se efectúa en la ficha "Modo de operación". Si se puede seleccionar un tiempo de retardo de DP suficientemente grande, éste es un procedimiento seguro para evitar interrupciones de la comunicación en el sistema maestro DP. El inconveniente de esta solución es que el tiempo de ciclo se hace más largo en general. (Nota: si se selecciona un tiempo de retardo suficientemente largo, se puede evitar así también el comportamiento antes descrito como "Interrupción de la comunicación DP al cargar enlaces FDL".)

 

6.1      Diagnóstico

 

 

6.1.1      Diagnóstico no paralelo a procesos de carga

El diagnóstico no debe estar en marcha mientras se desarrolla un proceso de carga a través de la configuración de hardware o dentro de la configuración de enlaces, pues en tal caso podría fracasar el proceso de carga.

 

6.1.2      Diagnóstico de Maestro DP / Cambiar conexiones

Si durante el diagnóstico en un maestro DP se suprime la conexión MPI del maestro DP y se conecta en otro módulo, no aparece ningún aviso. Por el contrario, se intenta continuar el diagnóstico del maestro DP. Recomendación: termine el diagnóstico antes de realizar cambios en la conexión del PG/PC. Inicie de nuevo el diagnóstico tras haber realizado el cambio en las conexiones.

 

6.2      Cargador de firmware

Su concesionario SIEMENS le puede proporcionar actualizaciones del firmware de CPs SIMATIC NET. NCM S7 contiene por ello un cargador de firmware. El manejo del mismo se explica en el archivo "Léame" que se adjunta al firmware.

Encontrará el cargador de firmware en el menú Inicio.  Inicio – Siemens Automation – SIMATIC – STEP7 – NCM S7 – Cargador de firmware

6.2.1        Módulos Ethernet autorizados

El cargador de firmware está autorizado para los siguientes módulos con Softnet S7/Controladores de PG:

·         Tarjeta NDIS Ethernet

·         CP 1613

 

6.2.2      Módulos PROFIBUS autorizados

El cargador de firmware está autorizado para los siguientes módulos:

·         CP 5611

·         MPI onboard

·         SIMATIC RI45 PII con CP5611 onboard

7         Problemas y subsanación de fallos

 

7.1      Aviso de error al cargar en un CP SIMATIC NET sin estar activada la interconexión en red

Al cargar una configuración de hardware para un CP SIMATIC NET en la que se haya desactivado la interconexión en red del CP, se presenta una aviso de error de protocolo S7 que indica que se ha destruido un objeto. Este aviso de error resulta del hecho de que el CP se debe iniciar de nuevo tras la carga, pero no le es posible reanudar su funcionamiento normal debido a la falta de interconexión en la red. El aviso de error se puede ignorar si la interconexión se ha desactivado intencionadamente. En otro caso se puede subsanar el fallo seleccionando la interconexión en red en el cuadro de diálogo de propiedades del CP.

 

7.2      Perturbaciones en la red MPI al cargar firmware

Al cargar el firmware de un CP en una estación SIMATIC 300 se pueden producir perturbaciones en la red MPI, si la CPU de la estación está conectada a una red MPI. Esto puede ser causa de perturbaciones en el tráfico de datos.

 

7.3      Problemas al imprimir la panorámica de enlaces

En la impresión de la panorámica de enlaces se utiliza para el ancho de columna el mismo ajuste que en el cuadro de diálogo. Si, por ejemplo, desea comprimir la columna destinada a un nombre hasta el punto que sólo se pueda ver una parte del nombre, éste también será visible sólo en parte en la copia impresa. Corrija en tal caso el ancho de columna en el cuadro de diálogo.

 

7.4      Lock / Unlock está habilitado para las siguientes CPU

 

CPU 412-1                    6ES7 412-1XF03-0AB0  HW versión 4 / SW versión V1.1.2

CPU 412-2                    6ES7 412-2XG00-0AB0  HW versión 4 / SW versión V1.1.2

CPU 414-2 con 128k     6ES7 414-2XG03-0AB0  HW versión 4 / SW versión V1.1.2

CPU 414-3                    6ES7 414-3XJ00-0AB0   HW versión 2 / SW versión V1.1.2

CPU414-4H                  6ES7 414-4HJ00-0AB0  HW versión 1

CPU 416-2 con 0,8 M    6ES7 416-2XK02-0AB0  HW versión 4 / SW versión V1.1.3

CPU 416-3                    6ES7 416-3XL00-0AB0  HW versión 2 / SW versión V1.1.2

CPU 417-4                    6ES7 417-4XL00-0AB0  HW versión 3 / SW versión V1.1.2

7.5      Interlocutor de enlaces E-Mail en otro proyecto

Para enlaces E-Mail no se puede indicar ningún interlocutor que se encuentre en un proyecto distinto.

 

7.6      Servidor OPC SNMP

Al cambiar de nombre variables ya instanciadas se tiene que introducir sólo el nombre del ítem, sin el punto ni las cifras siguientes.

A través de una WinCC Application configurada sólo se exportan a WinCC variables generales SCALANCE SNMP. Variables SNMP específicas del tipo no se exportan a WinCC ni se visualizan.

Los perfiles SNMP no se guardan ni archivan en el proyecto.

Solución: copiar manualmente los perfiles SNMP.

 

7.7      Cargar firmware con SIMATIC RI45 PII

Si en el caso de SIMATIC RI45 PII desea cargar un firmware a través de la interfaz MPI/DP en un CP PROFIBUS, tiene que instalar CP5611 en la herramienta de interfaces y que seleccionar CP5611(FWL) como parametrización de módulo.

 

7.8      Soporte de FMS-Broadcast

La función FMS-Broadcast se soporta a partir de las siguientes ediciones de CP y firmware:

CP 443-5 Basic             edición 8 y versión de firmware V2.20

CP 343-5                      edición 3 y versión de firmware V2.20

 

Atención: si se carga una configuración de FMS-Broadcast para ediciones menos recientes (de CP o firmware), se produce un comportamiento incorrecto del CP.

 

7.9      Importación de archivos FMS GSD

Si se importan repetidamente archivos FMS GSD de forma consecutiva sin finalización de NetPro, puede suceder que no se actualicen algunos de los parámetros.

Para evitar este problema debería reiniciar NetPro entre las importaciones repetidas.

 

7.10  Interlocutor de enlaces FMS en otro proyecto

Para enlaces FMS no se puede indicar ningún interlocutor que se encuentre en un proyecto distinto.

 

7.11  CP 443-1 EX 40 Advanced posible sólo con CPUs actuales

 

El CP 443-1 EX40 Advanced sólo se puede utilizar con CPUs actuales. Se excluye el funcionamiento con CPUs que ya no sean actuales.

En caso de funcionamiento individual de la H-CPU no se permite el uso de CP 443-1 EX40/EX41.

 

7.12  La CPU no puede acceder al maestro AS-i.

El mensaje "La CPU no puede acceder al maestro AS-i" puede aparecer en caso de telecarga ("upload") de un maestro AS-i.

Este mensaje se puede atribuir a las siguientes causas:

- La estación S7 no está configurada.

- El maestro AS-i no está accesible.

- El maestro AS-i es de un tipo incorrecto o de la versión incorrecta.

- El maestro AS-i está sobrecargado temporalmente por accesos a registros de datos.

 

7.13  "Asignar dirección Ethernet" no es compatible con CP 1613

El CP 1613 no da soporte a la función "Asignar dirección Ethernet“.

 

7.14  Visualización incorrecta de parámetros tras modificar la configuración y cargar a través de "Conexiones y pasos de red"

Si en NetPro se configura un módulo CP para otra red PROFIBUS y a continuación se carga el módulo en cuestión a través del comando de menú "...>Conexiones y pasos de red“, se ha de observar el siguiente comportamiento:

En NCM Diagnóstico aparecen en este caso, bajo los parámetros de bus PROFIBUS, todavía el nombre y el ID de la red PROFIBUS válida antes de la reconfiguración del módulo CP.

Solución: para obtener una visualización correcta se tiene que cargar el módulo a través del comando de menú "...>Estaciones marcadas“.

 

7.15  Recarga de "Conexiones y pasos de red" en la estación de PC SIMATIC

La estación de PC SIMATIC no puede desarrollar la recarga de conexiones y pasos de red si la estación se encuentra en el estado RUN.

En el caso de estaciones PC sólo se pueden recargar conexiones. Sólo el mensaje de error "El bloque de datos del sistema 1xxx no se ha podido borrar", que aparece al cargar, permite reconocer que aquí resultan afectados bloques de datos del sistema (SDB) de Routing.

 

7.16  El CP 343-1 Lean Device y la estación tienen el mismo nombre

Si el CP 343-1 Lean se utiliza como IO-Device, el CP puede estar archivado en el mismo proyecto. Si el Device y la estación han de ser iguales, tienen que coincidir las direcciones IP y el nombre del equipo en el IO-Device y en la estación central. Si los nombres no coinciden, no es posible el acoplamiento en servicio.

 

7.17  IE/AS-i Link PN IO 6GK1411-2AB10 y 6GK1411-2AB20

En la lista general de direcciones se muestra en la columna S - Slot (ranura) el número de fila y no el número de "slot".

 

7.18  IE/PB Link

Si en HW Config se entra en línea con un IE/PB Link en el ramal PNIO, aparece siempre el siguiente mensaje: Diferencias Online/Offline.

Si se contesta la pregunta "Utilizar no obstante la configuración almacenada para el diagnóstico" con OK ("Aceptar"), funciona el diagnóstico.

 

7.19  CP 1616/04

Un CP1616/04 que posea submódulos Interface y Port (p. ej. CP 1604 V2.1 PN 2.1) no debe ser sustituido por un CP que no posea tales submódulos (p. ej. CP1604 V2.1 PN 2.0).

Tras modificar las relaciones de transferencia se tienen que compilar de nuevo las estaciones, para que las relaciones de transferencia modificadas sean adoptadas también en la configuración (SDB’s).

 

7.20  CP 343-1 6GK7 343-1EX30-0XE0

Only one CP343-1 6GK7 343-1EX30-0XE0 IO-controller per rack is released.

It’s possible that already used default addresses are proposed when a device is added at the PNIO Bus.  

7.21  Connections establish connection timeout

Meaningful are only values from 2000ms.
