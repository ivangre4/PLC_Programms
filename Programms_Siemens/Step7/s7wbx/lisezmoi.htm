6.1      Notes concernant le fichier Lisezmoi

Avant d'utiliser NCM S7 lisez attentivement le fichier Lisezmoi.  

Le fichier Lisezmoi contient des informations de dernière heure sur le produit. Ces dernières complètent les instructions des manuels.

 

Le fichier Lisezmoi est accessible après l'installation par le menu Démarrer. Démarrer – SIMATIC – Notes sur le produit – NCM S7 – lisezmoi.htm.

 Note that the software in MS Windows Vista is no longer under "Start menu > SIMATIC"; it is now located under "Start menu > Program Files > Siemens Automation > SIMATIC". This was made necessary by the current architecture of MS Windows Vista. The information in the documentation and online help must be interpreted accordingly.

 

6.2      Définition de la configuration matérielle

 

6.2.1      Configuration du mode maître DP

 

6.2.1.1  Système maître DP sur CP 342-5 maître DP

Si, sur le CP 342-5 DP fonctionnant en mode "maître DP", vous avez défini un système maître DP avec un esclave DP sans module, le CP ne pourra pas être démarré. Vous devez par conséquent configurer sur ce maître DP un module pour chaque esclave DP configuré !

 

6.2.1.2  CP342-5 comme esclave DP avec LSAP 56

Le LSAP 56 est réservé sur un esclave DP à la fonction "Read_Inputs : Lecture des données d’entrée par le maître Kl2". Il n’est alors pas admissible de configurer en même temps une liaison FDL Broadcast ou Muticast avec LSAP 56

6.2.1.3  Interruption de la communication DP lors du chargement de liaisons FDL

Si vous chargez des liaisons FDL pendant le fonctionnement du système (état RUN), les communications DP seront interrompues pendant la durée du chargement. Tenez-en compte pour les applications DP à temps critique ! (Voir également à ce propos la note sous "Remède" au point suivant.

 

6.2.1.4  Profil de bus à temps critique lors de l'utilisation parallèle de liaisons DP et FDL

Si vous utilisez simultanément des liaisons FDL et des systèmes maître DP sur votre sous-réseau ROFIBUS (L2), il se peut que le profil de paramétrage de bus "Standard" soit à l'origine d'un comportement à temps critique dans les systèmes maître DP. En présence d'un taux d'utilisation important du bus dû à des transmissions via liaisons FDL, il se peut que des esclaves DP se déconnectent sporadiquement suite au dépassement du délai de scrutation défini sur le maître DP. Ce phénomène peut influencer le comportement temps réel d'applications à temps critique.

 

Remède :

Option 1 : Utilisez le profil "Universel".

Option 2 : Augmentez  la temporisation DP dans le dialogue des propriétés du CP. Vous élargissez ainsi le créneau de temps mis à la disposition de la communication FDL. Le paramétrage s'effectue dans l'onglet "Mode". Si vous pouvez spécifier une temporisation DP suffisamment longue, vous disposerez là du meilleur moyen pour éviter des interruptions de la communication au sein du système maître DP. L'inconvénient de cette solution réside dans les temps de cycle qui, d'une manière générale, seront plus longs. (Nota : En choisissant une temporisation suffisamment longue vous pouvez également éliminer le comportement décrit plus haut de "l'interruption de la communication DP lors du chargement de liaisons FDL".)

 

6.3      Diagnostic

 

 

6.3.1      Diagnostic non parallèle aux opérations de chargement

Ne démarrez pas le diagnostic pendant le chargement de la configuration matérielle ou pendant la configuration des liaisons. L'opération de chargement risque sinon d'échouer.

 

6.3.2      Diagnostic maître DP / Modification des connexions

Si vous supprimez la connexion MPI d'un maître DP pendant le diagnostic et que vous embrochez un autre module, aucun message ne s'affichera ! Le système tentera au lieu de cela de poursuivre le diagnostic du maître DP. Recommandation : Fermez le diagnostic avant de modifier la connexion du PG/PC. Relancez le diagnostic après avoir modifié la connexion.

 

6.4      Chargeur de firmware

Vous pouvez vous procurer les mises à jour du firmware des CP SIMATIC auprès de votre agence SIEMENS. NCM S7 est doté à cette fin d'un chargeur de firmware. Le fichier Lisezmoi qui accompagne le firmware indique comment utiliser le chargeur de firmware. 

Vous trouverez le chargeur de firmware dans le menu Démarrer.  Démarrer – Siemens Automation – SIMATIC – STEP7 – NCM S7 – Chargeur de firmware 

 

6.4.1        Modules Ethernet validés

Le chargeur de firmware est validé pour les modules suivants dotés de pilotes Softnet S7/PG :

 

·         Carte Ethernet NDIS

·         CP 1613

 

 

6.4.2      Modules PROFIBUS validés

Le chargeur de firmware est validé pour les modules suivants :

 

·         CP 5611

·         MPI onboard

·         SIMATIC RI45 PII avec CP5611 onboard

7         Problèmes et remèdes

 

7.1      Message d'erreur lors du chargement sur CP SIMATIC NET dont la connexion au réseau est désactivée

Si vous chargez la configuration matérielle d'un CP SIMATIC NET dont la connexion au réseau a été désactivée, un message d'erreur de protocole S7 signalant qu'un objet a été détruit, s'affiche. Ce message d'erreur provient du fait, qu'après un chargement, le CP devrait être redémarré alors qu'il ne peut pas le faire puisque sa connexion au réseau a été désactivée. Vous pouvez ignorer ce message d'erreur si vous avez désactivé la connexion au réseau intentionnellement. Vous pouvez sinon remédier au défaut en activant la connexion au réseau dans le dialogue des propriétés du CP.

 

7.2      Problèmes sur le réseau MPI lors du chargement du firmware

Des incidents peuvent survenir sur le réseau MPI lors du chargement du firmware d'un CP équipant une station SIMATIC 300 si la CPU de la station est connectée à un réseau MPI. Il peut en résulter des problèmes lors de l'échange de données.

 

 

 

7.3      Problèmes lors de l'impression de la liste des liaisons

Lors de l'impression de la liste des liaisons, la largeur de colonne utilisée est la même que celle spécifiée dans le dialogue . Si, après avoir réduit la largeur de la colonne des noms, vous ne voyez plus qu'une partie du nom, le nom sera également tronqué à l'impression. Rectifiez dans ce cas la largeur de colonne dans le dialogue.

7.5      Liaisons E-Mail - Partenaire dans un autre projet

Dans le cas de liaisons E-Mail, vous ne pouvez pas spécifier de partenaires dans un autre projet.

 

7.6      Serveur OPC SNMP

Lorsque vous renommez des variables déjà instanciées, il suffit de spécifier le nom d'item - sans le point et les chiffres qui suivent.

L’application WinCC configurée ne permet d’exporter vers WinCC que des variables SCALANCE SNMP générales. Les variables SNMP spécifiques ne sont pas affichées vers WinCC, ni affichées.

Les profils SNMP ne sont pas enregistrés ni archivés dans le projet.

Remède : copiez les porfils SNMP manuellement.

 

7.7      Chargement de firmware avec RI45 PII SIMATIC

 

Si, dans le cas du RI45 PII SIMATIC, vous voulez charger le firmware sur un CP PROFIBUS via l'interface MPI/DP, installez le CP5611 dans l'outil d'interface puis sélectionnez comme paramétrage de module CP5611(FWL).

 

7.8      Prise en charge du broadcast FMS

La fonction broadcast FMS est prise en charge à partir des versions de CP et de firmware suivantes :

CP 443-5 Basic             Version matérielle 8 et Version logicielle V2.20

CP 343-5                      Version matérielle 3 et Version logicielle V2.20

 

Attention ! Si vous chargez une configuration FMS Broadcast sur une ancienne version (de CP ou logiciel), le CP ne fonctionnera pas correctement.

 

7.9      Importation de fichiers GSD FMS

Si vous importez plusieurs fois de suite des fichiers GSD FMS sans clore NetPro, il se peut que certains paramètres ne soient pas actualisés.

Pour éviter ce problème des importations multiples, redémarrez NetPro après chaque importation.

 

7.10  Liaisons FMS - Partenaire dans un autre projet

Dans le cas de liaisons FMS, vous ne pouvez pas spécifier de partenaires dans un autre projet.

 

7.11  CP 443-1 EX 40 Advanced uniquement utilisable avec une CPU récente

Le CP 443-1 EX40 Advanced n'est utilisable qu'avec des CPU récentes. L'utilisation avec d'ancienne CPU est exclue.

L’utilisation des CP 443-1 EX40/EX41 n’est pas autorisée en mode de fonctionnement autonome de la CPU H.

 

7.12  The CPU cannot reach the AS-i master.

The message “CPU cannot reach the AS-I master” can be distributed at the upload of an AS-I master.

 

The report can be explained by the following causes:

·         The S7 station isn't configured.

·         The AS-i master isn't obtainable.

·         The AS-i master is of the wrong type or has the wrong version.

·         The AS-i master is overloaded by data set accesses temporarily.

  

7.13  „Assign Etnernet Adress“ over CP 1613 not possible

The function " Assign Etnernet Adress " isn't supported over CP 1613.

 

7.14  Affichage de paramètres erroné après modification de la configuration et chargement via "Liaisons et passerelles"

Affichage de paramètres erroné après modification de la configuration et chargement via "Liaisons et passerelles"

Si vous configurez sous NetPro un module CP pour un autre réseau PROFIBUS et si vous chargez ensuite le module en question à l'aide de la commande de menu "..>Liaisons et passerelles", tenez compte du comportement suivant :

NCM Diagnostic affiche encore sous les paramètres de bus PROFIBUS le nom et l'ID du réseau PROFIBUS qui étaient valables avant la reconfiguration du module CP.

Remède : Pour obtenir un affichage correct, chargez le module au moyen de la commande de menu "...>Stations sélectionnées" !

 

7.15  Reloading "Connections and Gateways“ on the SIMATIC PC Station

The SIMATIC PC station cannot handle the reloading of connections and gateways while the station is in RUN. Only connections can be reloaded on PC stations.

You can only recognize that routing SDBs are involved here when the error message "System data block 1xxx could not be deleted" appears during loading.

 

7.16  Même nom pour le CP 343-1 Lean Device et la staion

Si le CP 343-1 Lean est utilisé comme IO Device, le CP peut être enregistré dans le même projet. Si le Device et la station sont identiques, les adresses IP et le nom d’appareil sur l’IO Device et la station centrale doivent être identiques.

Si ce n’est pas le cas, le couplage en fonctionnement ne peut pas avoir lieu.

 

7.17  IE/AS-i Link PN IO 6GK1411-2AB10 et 6GK1411-2AB20

Dans le récapitulatif des adresses, la colonne S (Slot) des emplacements affiche le numéro de rangée et non pas le numéro d’emplacement.

7.18  IE/PB Link

Lorsqu’on passe en ligne sous HW Config avec un IE/PB Link sur le segment PNIO, le message suivant apparaît régulièrement : Différences entre en ligne/hors ligne

Si l’on répond à la question : Voulez-vous tout de même utiliser la configuration enregistrée pour le diagnostic ? par OK, le diagnostic fonctionne.

 

7.19  CP 1616/04

Un CP1616/04 qui possède des sous-modules d'interface et de port (CP 1604 V2.1 PN 2.1 p. ex.) ne doit pas être échangé contre un CP qui n'est pas équipé de ces sous-modules (CP1604 V2.1 PN 2.0 p. ex.).

Les stations doivent être à nouveau compilées après modification des relations de transfert afin que les relations de transfert modifiées puissent être prises en compte dans la configuration (SDB).

 

7.20  CP 343-1 6GK7 343-1EX30-0XE0

Only one CP343-1 6GK7 343-1EX30-0XE0 IO-controller per rack is released.

It’s possible that already used default addresses are proposed when a device is added at the PNIO Bus.  

7.21  Connections establish connection timeout

Meaningful are only values from 2000ms.
