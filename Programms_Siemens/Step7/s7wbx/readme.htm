6.2      Project Engineering of the Hardware Configuration

 

6.2.1      Project Engineering for the DP Master Mode

 

6.2.1.1  DP Master System with the CP 342-5 as DP Master

If the CP 342-5 DP is operating in the "DP master mode" and the DP master was defined with a DP slave without submodule, it is not possible to start the CP. You must therefore configure at least one submodule for every configured DP slave for this DP master!

 

6.2.1.2         CP342-5 as DP Slave with LSAP 56

With a DP slave, LSAP 56 is occupied by the "Read_Inputs by master class 2" function. It is then no longer permitted to configure an FDL broadcast or multicast connection with LSAP 56 at the same time.

 

6.2.1.3  Interrupting DP Communication when Downloading FDL Connections

If FDL connections are downloaded during system operation (RUN), DP communication is interrupted for this time. Keep this in mind with time-critical DP applications! (See also "Remedy" in the next section.)

 

6.2.1.4  Time-Critical Bus Profile and Parallel Operation of DP and FDL Connections

If you operate FDL connections and DP master systems on your PROFIBUS(L2) subnet at the same time, the "Standard" bus parameter profile can lead to time-critical behavior in the DP master systems. If there is heavy traffic due to transmissions on FDL connections, DP slaves can sporadically drop out of the transmission because the monitoring time set on the DP master times out. This can influence the real-time response of a time-critical application.

 

Remedy:

Option 1: Use the "Universal" profile.

Option 2: Increase the DP delay time in the properties dialog of the CP. This extends the time window available for FDL communication. You make the setting in the "Operating Mode" tab. If the DP delay time can be selected high enough, this is a reliable way of avoiding interruptions in communication in the DP master system. The disadvantage of this solution is generally the high cycle time. (Note: If the delay time is selected adequately high, the response described above "Interrupting DP Communication when Downloading FDL Connections" can be eliminated.)

 

6.3      Diagnostics

 

6.3.1      No Diagnostics Parallel to Downloading

Diagnostics should not be started while a download is taking place during hardware configuration or during project engineering of connections. The download can otherwise fail.

 

6.3.2      DP Master Diagnostics / Changing Attachments

If the MPI attachment is removed from the DP master and connected to another module during diagnostics, no message is output! On the contrary, the system attempts to continue DP master diagnostics. Recommendation: Stop diagnostics before you make changes to the PG/PC attachment. After changing the attachment, start diagnostics again.

7         Problems and Troubleshooting

 

7.1      Error Message when Downloading to a SIMATIC NET CP without Activated Networking

When you download a hardware configuration for a SIMATIC NET CP in which the networking of the CP was deactivated, this results in an S7 protocol error message that indicates that an object was destroyed. This error message results from the fact that the CP should be restarted but as a result of the lack of networking, it cannot go into normal operation again. The error message can be ignored if networking was deliberately deselected. Otherwise the error can be eliminated by selecting networking in the properties dialog of the CP.

 

7.2      Problems on MPI Network when downloading Firmware

When loading the firmware of a CP in a SIMATIC 300 station, problems can occur on the MPI network when the CPU of the station is connected with an MPI network. This can lead to problems in the data traffic.

 

7.3      Problems Printing the Connection Overview

When printing the connection overview, the same setting for the column width is used as in the dialog.  If, for example, you reduce the size of a column so that only part of the name is visible, only part of the name will be printed out. In this case, correct the column width in the dialog.

7.5      E-Mail Connections - Partner in other Project

On E-mail connections, no partner can be specified in the other project.

 

7.6      OPC Server SNMP

When renaming variables that are already instantiated, you only need to enter the item name        without the period and the following numbers.

 

Using a configured WinCC application, only general SCALANCE SNMP variables are exported to WinCC. Type-specific SNMP variables are not exported to WinCC and not displayed.

 

SNMP profiles are not saved and archived in the project.

Remedy: Copy SNMP profiles manually.

 

7.7      Loading Firmware with SIMATIC RI45 PII

With the SIMATIC RI45 PII, if you want to download firmware over the MPI/DP interface to a PROFIBUS CP, you must install the CP 5611 in the interface tool and select CP5611(FWL) as the module parameter assignment.

 

7.8      FMS Broadcast Support

The FMS Broadcast functionality is supported as of the following CP and firmware versions:

CP 443-5 Basic             version 8 and firmware version V2.20

CP 343-5                      version 3 and firmware version V2.20

 

Note! If an FMS broadcast project engineering configuration is downloaded to older versions (CP or firmware), the CP will malfunction.

 

7.9      Importing FMS GSD Files

If FMS GSD files are imported several times one after the other without closing NetPro, it is possible that individual parameters are not updated.

To avoid this problem, you should restart NetPro between imports.

 

7.10  FMS Connections - Partner in other Project

On FMS connections, no partner can be specified in the other project.

 

7.11  Operating CP 443-1 EX 40 Advanced only with Current CPUs

The CP 443-1 EX40 Advanced can only be operated with the current CPUs. Operation with CPUs that are no longer up to date is not possible.

 

If the H-CPU is in single mode, the use of a CP 443-1 EX40/EX41 is not permitted

 

7.12  The CPU cannot reach the AS-i master.

The message “CPU cannot reach the AS-I master” can be distributed at the upload of an AS-I master.

 

The report can be explained by the following causes:

·         The S7 station isn't configured.

·         The AS-i master isn't obtainable.

·         The AS-i master is of the wrong type or has the wrong version.

·         The AS-i master is overloaded by data set accesses temporarily.

 

7.13  "Assign Ethernet Address" over CP 1613 not possible

The function "Assign Ethernet Address" is not supported over CP 1613.

 

7.14  Incorrect parameter display after changed configuration and download with "Connections and Gateways"

If you configure a CP module for a different PROFIBUS network in NetPro and then download the module with the menu command "...>Connections and Gateways", note the following response:

In this situation, the name and network ID of the PROFIBUS network valid before the reconfiguration of the CP module still appears among the PROFIBUS bus parameters in NCM diagnostics.

Remedy: To obtain a correct display, please download the module with the "...>Selected Stations" menu command!

 

7.15  Reloading "Connections and Gateways" on the SIMATIC PC Station

The SIMATIC PC station cannot handle the reloading of connections and gateways while the station is in RUN. Only connections can be reloaded on PC stations. You can only recognize that routing SDBs are involved here when the error message "System data block 1xxx could not be deleted" appears during loading.

 

7.16  CP 343-1 Lean Device and Station with the same Name

If the CP 343-1 Lean is used as an IO device, the CP can be stored in the same project. If you want the device and the station to be the same, the IP addresses and the device name on the IO device and in the central station must match. If the names do not match, no link is possible during operation.

 

7.17  IE/AS-i Link PN IO 6GK1411-2AB10 and 6GK1411-2AB20

In the address overview, the S column (Slot) shows the row number and not the slot number.

 

7.18  IE/PB Link

If you go online with an IE/PB Link on the PNIO line in HW Config, the following message is always displayed: Differences online/offline

If you answer the prompt: "You can, however, use the saved configuration for diagnosis" with OK, diagnostics will work correctly.

 

7.19  CP 1616/04

A CP1616/04 that has interface and port submodules (for example a CP 1604 V2.1 PN 2.1) must not be replaced with a CP that does not have these submodules (for example a CP 1604 V2.1 PN 2.0).

After changing the transfer relations, stations must be recompiled so that the changed transfer relations are also included in the configuration (SDBs).

7.20  CP 343-1 6GK7 343-1EX30-0XE0

Only one CP343-1 6GK7 343-1EX30-0XE0 IO-controller per rack is released.

It’s possible that already used default addresses are proposed when a device is added at the PNIO Bus.  

7.21  Connections establish connection timeout

Meaningful are only values from 2000ms.
