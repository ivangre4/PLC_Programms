6.2      硬件配置的项目工程

 

6.2.1      DP主站模式的项目工程

 

6.2.1.1  以CP 342-5作为DP主站的DP主站系统

如果CP 342-5 DP运行在“DP主站模式”，并且DP主站被定义为带一个没有子模块的DP从站，那么将不能启动CP。因此，必须为该DP主站的每一个已组态的DP从站组态至少一个子模块！

 

 

6.2.1.2  在下载FDL连接时中断DP通讯

如果在系统运行(RUN)期间下载FDL连接，在这期间，DP通讯将会中断。使用对时间要求苛刻的DP应用程序时请牢记这一点！(参见下一节的“纠正方法”。)

 

6.2.1.3  对时间要求苛刻的总线配置文件和DP与FDL连接的并行操作

如果您在PROFIBUS(L2)子网上同时操作FDL连接和DP主站系统，“标准的”总线参数配置文件可能会导致在DP主站系统中对时间要求苛刻的运行状况。如果由于FDL连接上的传送导致严重的阻塞，DP从站会因为超出DP主站上设置的监视时间而间歇性地退出传送。这会影响到对时间要求苛刻的应用程序的实时响应。

 

纠正方法：

方法1：使用“通用”配置文件。

方法2：增加  CP属性对话框中的DP延时时间。这可以扩充FDL通讯可用的时间窗口。您可在“工作模式”标签中进行设置。将DP延时时间设得足够的高，这不失为一种避免DP主站系统通讯中断的可靠方法。但该方案的缺点是往往使得周期变得很长。(注意：如果选择了适当高的延时时间，前文描述的  “在下载FDL连接时中断DP通讯”响应就可消除。)

 

6.3      诊断

 

 

6.3.1      没有与下载并行的诊断

在硬件配置或连接的项目工程期间，不能在下载正在进行时启动诊断。否则，下载可能会失败。

 

6.3.2      DP主站诊断/更换附件

如果在诊断期间将MPI附件从DP主站上拆除并连接到其它的模块，将不会输出任何消息！系统反而还将试图继续进行DP主站诊断。建议：在改变PG/PC附件前停止诊断。在附件改变之后，再次启动诊断。

7         故障和故障诊断

 

7.1      在没有激活的联网时下载到SIMATIC NET CP的出错消息

当您为一个没有激活CP联网的SIMATIC NET CP下载硬件组态时，将会导致一条S7协议错误消息，指示对象已损坏。出错消息源于应该重新启动的CP没有联网，从而不能再次进入正常操作。如果是有意取消选定联网的，可以忽略出错消息。另外，可在CP属性对话框中选择联网来消除错误。

 

7.2      在下载硬件时MPI网络上的故障

在SIMATIC 300站上装载CP的硬件时，如果该站的CPU连接着MPI网络，则MPI网络可能会出错。这可能会导致数据通讯量故障。

 

7.3      打印连接总览时的故障

当打印连接总览时，对话框中使用了同样的的列宽设置。  这样的话，如果减少列宽，名称将不能完整显示，  就只能打印名称的一部分。这种情况下，需要在对话框中更正列宽。

7.5      电子邮件连接 - 其它项目中的伙伴

在电子邮件连接上，不能在其它项目中指定对象。

 

7.6      OPC服务器SNMP

当重命名已经初始化的变量时，只需输入它的条目名称，不必输入点号和跟随的数字。

 

7.7      使用SIMATIC RI45 PII装载硬件

对于SIMATIC RI45 PII，如果您想通过MPI/DP接口将硬件下载到PROFIBUS CP，您必须在接口工具中安装CP 5611，并选择CP5611(FWL)作为模块参数分配。

 

7.8      FMS广播支持

从下列CP和硬件版本起开始支持FMS Broadcast功能：

CP 443-5基本                 版本8和硬件版本V2.20

CP 343-5                      版本3和硬件版本V2.20

 

注意！如果将FMS广播项目工程组态下载给旧的版本(CP或硬件)，CP将会出故障。

 

7.9      导入FMS GSD文件

如果在没有关闭NetPro的情况下，接连导入多次FMS GSD文件，可能出现单个参数没有更新的现象。

为了避免出现该问题，应当在每两次导入之间重新启动NetPro。

 

7.10  FMS连接 - 其它项目中的伙伴

在FMS连接上，不能在其它项目中指定任何伙伴。

 

7.11  只使用当前CPU操作CP 443-1 EX 40高级版

CP 443-1 EX40高级版只能使用最新的CPU操作。不使用最新的CPU将不能进行操作。

 

7.12  CPU无法访问AS-i主站。

在上传AS-I主站时可能会出现消息“CPU无法访问AS-I主站”。

 

导致出现该报告原因如下：

・         没有组态S7站。

・         不能获取AS-i主站。

・         AS-i主站的类型错误或版本错误。

・         AS-i主站暂时因数据集访问而超载。

 

 

7.13  通过CP 1613“设置以太网地址”不可行

CP 1613不支持功能“分配以太网地址”。

 

7.14   载入“连接和网关”后显示错误的PB参数。

如果在NetPro中在另一个Profibus网络上组态了一个CP组件，然后通过菜单选项“连接和网关”将其载入，那么在NCM诊断中的Profibus参数下就会显示Profibus网络的名称和网络标识符，该网络在CP组件更改组态之前已选中。

为了获得正确的显示，应通过菜单选项“已标记的站”装入组件！

7.15  在SIMATIC PC站上重新装载"连接和网关"

SIMATIC PC站在RUN模式下不能处理连接和网关的重新装载任务。只能在PC站上重新装载连接。仅当在装载期间显示出错消息"无法删除系统数据块1xxx" 时，才能识别出此处与路由SDB有关。

 

7.16  CP 343-1 Lean设备和站同名

如果CP 343-1 Lean用作IO设备，则CP可以存储在同一项目中。如果您希望设备和站为相同名称，则IO设备的IP地址和设备名称必须与中央站的相匹配。如果名称不匹配，则在运行期间不能建立链接。

 

7.17  IE/AS-i Link PN IO 6GK1411-2AB10和6GK1411-2AB20

在地址总览中，S列(插槽)显示的是行号而非插槽号。

 

7.18  IE/PB Link

如果您在HW Config中的PNIO线上持续使用IE/PB Link，则将终显示以下消息：在线/离线差异

对于提示："但是，您可以使用已保存的组态进行诊断" ，如果您回答"是"，则诊断将正常进行。

 

7.19  CP 1616/04

请勿将带有接口和端口子模块的CP1616/04 (例如CP 1604 V2.1 PN 2.1)用不具备这些子模块的CP (例如CP 1604 V2.1 PN 2.0)代替。

更改传送关系后，必须对站进行重新编译，以便更改后的传送关系也包含在组态(SDB)中。

7.20  CP 343-1 6GK7 343-1EX30-0XE0

在每个机架上只能释放一个CP343-16GK7 343-1EX30-0XE0 IO控制器。

It’s possible that already used default addresses are proposed when a device is added at the PNIO Bus.  

7.21  Connections establish connection timeout

Meaningful are only values from 2000ms.
